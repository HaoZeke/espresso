23 /** \file binary_file_tcl.hpp
24  This file defines a binary file format for the particle data.
25 
26 The functions in this file are no longer supported by the Espresso
27 team. Use them at your own risk!
28 
29  It is the header file for \ref binary_file_tcl.cpp "binary_file_tcl.c" and provides
30  functions to read and write binary particle data to Tcl channels.
31  It also defines the structures and constants necessary to interprete/
32  generate particle data files in this format. THE FILE FORMAT IS HARDWARE
33  DEPENDENT, SINCE RAW DATA TYPES ARE WRITTEN!!!
34 
35  <p>
36  The file format conists of the following:
37  <ol>
38  <li> \ref MDHeader with \ref MDHeader::magic set to \ref MDMAGIC ("MD01") without trailing 0.
39  <li> then \ref MDHeader::n_rows chars out of
40  \ref POSX,\ref POSY,\ref POSZ,\ref VX,\ref VY,\ref VZ,\ref FX,\ref FY,\ref FZ,
41  \ref Q,\ref TYPE which determine the data that is contained in the following and
42  correspond to the respective fields of \ref Particle.
43  <li> now follows the particle data, always starting with one integer giving the
44  particles identity (this row is NOT defined before!), then the data for the fields
45  as defined before. The data types
46  for \ref POSX,\ref POSY,\ref POSZ,\ref VX,\ref VY,\ref VZ,\ref FX,\ref FY,\ref FZ,
47  \ref Q are double, for \ref TYPE integer.
48  <li> The file is finished with a single integer -1 (replacing the particle identity
49  of a subsequent particle).
50  </ol>
51 */
