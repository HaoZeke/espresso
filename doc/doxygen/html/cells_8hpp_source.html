23 /** \file cells.hpp
24  This file contains everything related to the cell structure / cell
25  system.
26 
27  The cell system (\ref Cell Structure) describes how particles are
28  distributed on the cells and how particles of different cells
29  (regardless if they reside on the same or different nodes)
30  interact with each other. The following cell systems are implemented:
31 
32  <ul>
33  <li> domain decomposition: The simulation box is divided spatially
34  ino cells (see \ref domain_decomposition.hpp). This is suitable for
35  short range interctions.
36  <li> nsquare: The particles are distributed equally on all nodes
37  regardless their spatial position (see \ref nsquare.hpp). This is
38  suitable for long range interactions that can not be treated by a
39  special method like P3M (see \ref p3m.hpp).
40  <li> layered: in x and y directions, it uses a nsquared type of interaction calculation,
41  but in z it has a domain decomposition into layers.
42  </ul>
43 
44  One can switch between different cell systems with the tcl command
45  cellsystem implemented in \ref cells.cpp .
46 
47  Some structures are common to all cell systems:
48 
49  <ul>
50  <li> All cells, real cells as well as ghost cells, are stored in the array \ref cells::cells with size \ref
51  n_cells. The size of this array has to be changed with \ref realloc_cells.
52  <li> Their are two lists of cell pointers to acces particles and
53  ghost particles on a node: \ref local_cells contains pointers to
54  all cells containing the particles physically residing on that
55  node. \ref ghost_cells contains pointers to all cells containing
56  the ghost particles of that node. The size of these lists has to be
57  changed with \ref realloc_cellplist
58  <li> An example using the cell pointer lists to access particle data
59  can be found in the function \ref
60  print_local_particle_positions. DO NOT INVENT YOUR OWN WAY!!!
61  </ul>
62 */

240 
241  If bit CELL_FLAG_FAST is set, the routine should try to save time.

245
