23 /** \file nsquare.hpp
24  This file contains the code for a simple n^2 particle loop.
25 
26  The nsquare cell system performs a full n^2 particle interaction
27  calculation over the simulation box. Therefore every node just
28  has a single cell containing all local particles plus one ghost
29  cell per other node. The communication is done via broadcasts
30  (exchange_ghosts and update_ghosts) and reduce operations
31  (collect_ghost_force).
32 
33  The algorithm used for interaction calculation is parallelized,
34  but a full communication is needed every time step. Let us assume
35  that the number of nodes P is odd. Then a node p will do the
36  interaction calculation with another node q iff \f$(q-p)\,mod\,P\f$ is even.
37  Of course then every node has to do the same amount of work
38  (provided the particles are distributed equally), and each
39  interaction pair is done exactly once, since for odd P \f$r\,mod\,P\f$
40  odd iff \f$-r\,mod\,P\f$ is even. For an even number of nodes,
41  a virtual additional processor is assumed, with which no interactions occur.
42  This means, that each communication cycle, 1 processor is idle, which is
43  pretty ineffective.
44 
45  The second main part of this cell system is a load balancer which
46  at the beginning of the integration is called and balances the
47  numbers of particles between the nodes. This means that the number
48  of particles per node should be around \f$N/P\f$, so the goal is to
49  let every processor have a number of particles which is one of the
50  two integers closest to \f$N/P\f$. The algorithm is greedy, i. e. it
51  searches for the node with most and least particles and transfers
52  as much as possible particles between them so that at least one of
53  them satisfies the constraints. Of course the algorithm terminates
54  if both satisfy the condition without transfer.
55 
56  The calculations themselves are just simple loops over all
57  appropriate particle pairs.
58 */
