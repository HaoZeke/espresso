19 /* * @File Header file for the correlation class
20  *
21  * This module allow to compute correlations (and other two time averages) on
22  * the fly an from files.
23  *
24  * The basic idea is that the user can write arbitrary function A and B that
25  * can depend on e.g. particle coordinates or whatever state of the MD box.
26  * These function can be vector valued, always indicated by dim_A and dim_B.
27  *
28  * The way they can be correlated is can be formulated as a (vector valued)
29  * operation on A and B. One example would be to calculate the product
30  * component by component, and if A and B are both the particle velocities,
31  * then one would obtain
32  * { <v_1x v_1x(t)> <v_1y v_1y(t)> <v_1z v_1z(t)> <v_2x v_2x(t)> <v_2y v_2y(t)> <v_2z v_2z(t)> ... }
33  *
34  * The idea of the algorithm is not to keep all As and Bs in memory, but
35  * feed the algorithm (or the container object) successively by new As and Bs
36  * and new correlation estimates are calculated on the fly. Those As and Bs
37  * which have exceeded a certain age are (first compressed, see below) and then
38  * discarded.
39  *
40  * To save memory, increase statistics and make the calculation possible for
41  * many orders of magnitude in time, the blocking algorithm in Frenkels book (p.91)
42  * is applied. Thus not all As and Bs of the whole "past" are stored but
43  * some of them are blocked. In this implementation always a blocking based
44  * on 2 is applied: All As and Bs not older than a certain tau_lin are stored
45  * as they were, those which are older are not entire stored, but only
46  * their compressed average value. All As and Bs older than 2*tau_lin
47  * are compressed in blocks of four etc.
48  *
49  * This leads to a hiearchical "history": on level 0 the last tau_lin values
50  * are stored. This is done in a cyclic array: The newest is always appended
51  * at the end of the array, and if the array length is reached, values
52  * are appended at the beginning, overwriting older values. We therefore
53  * have to carry the index newest[0] which tells, what is the index of the
54  * newest value of A and B.
55  *
56  * As soon as the first row of As and Bs is full, two of them are
57  * compressed by calculating the arithmetic mean, and stored in the
58  * first element on level 1. Then we can overwrite these two values
59  * on level 0, because we have saved them. Always if necessary
60  * we produce space on level 0 by compressing to level 1. On
61  * level 1 also an array with tau_lin entries is available to store
62  * the level-1-compresssed values. It is sucessivly filled
63  * and also cyclic. When it is filled, the values are stored
64  * on level 2 and so on.
65  *
66  * This allows to have a "history" over many orders of magnitude
67  * in time, without the full memory effort.
68  *
69  * Correlations are only calculated on each level: For
70  * tau=1,2,..,tau_lin the values are taken from level 1.
71  * For tau=tau_lin, tau_lin+2, .., 2*tau_lin we take the values
72  * from level 2. On level 2 we halso have values for 0,..tau_lin-2,
73  * but these are discared as we have "better" estimates on level 1.
74  *
75  * The functions A and B can get extra arguments. This can e.g. be an
76  * integer describing to the "type" of the particles to be considered,
77  * or in case of files, it is a file_data_source pointer, which tells
78  * the function from where to read. These arguments have to be a single
79  * pointer to an object that carries all relevant information
80  * to obtain A and B (from the MD Box or somewhere else).
81  *
82  * The correlation has to be initialized with all necessary information,
83  * i.e. all function pointers, the dimensions of A and B and their dimensions etc.
84  * When new A and B values should be processed, a single call of
85  * double_correlation_get_data(..) with the correlation object as an
86  * argument is enough to update A and B and the correlation estimate.
87  *
88  * Eventually the correlation can be printed.
89  *
90  *
91  * There is a lot of stuff to do:
92  * ==============================
93  * -> Write input functions that take TCL arrays for A and B to
94  * make the method available for TCL-coded variables
95  * -> Expand the file_data_source so that one can specify which
96  * columns of the file are to be processed
97  * -> Write more correlation operations (scalar product)
98  * -> Write more observable
99  * -> calculate an estimate of average values. This might be
100  * even necessary to calculate <(A-<A>)(B(tau)-<B>), which
101  * is often probably what people want
102  * -> Use the A_args to calculate As and Bs only for particular
103  * particle types (especially and example, so that other people can follow)
104  * -> Use the A_args to calculate molecular stuff in combination with
105  * the topology concept
106  * -> Tidy up the parsers (just a bit)
107  * -> Write some nice samples
108  * -> Write a destructor
109  * -> Finally: write the users guide
110  *
111  *
112  */

137  * it starts again from the beginning.
138  */
