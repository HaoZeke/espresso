23 /** \file communication.hpp
24  This file contains the asynchronous MPI communication.
25 
26  It is the header file for \ref communication.cpp "communication.c".
27 
28  The asynchronous MPI communication is used during the script
29  evaluation. Except for the master node that interpretes the Tcl
30  script, all other nodes wait in mpi_loop() for the master node to
31  issue an action using mpi_call(). \ref mpi_loop immediately
32  executes an MPI_Bcast and therefore waits for the master node to
33  broadcast a command, which is done by mpi_call(). The request
34  consists of three integers, the first one describing the action
35  issued, the second and third an arbitrary parameter depending on
36  the action issued. If applicable, the second parameter is the node
37  number of the slave this request is dedicated to.
38 
39  To add new actions (e. g. to implement new Tcl commands), do the
40  following:
41  - write the mpi_* function that is executed on the master
42  - write the mpi_*_slave function
43  - Add your slave function to CALLBACK_LIST in communication.cpp
44 
45  After this your procedure is free to do anything. However, it has
46  to be in (MPI) sync with what your new mpi_*_slave does. This
47  procedure is called immediately after the broadcast with the
48  arbitrary integer as parameter. To this aim it has also to be
49  added to \ref CALLBACK_LIST "callbacks". Last but not least for
50  debugging purposes you can add a nice name to \ref #names in the
51  same way. */
